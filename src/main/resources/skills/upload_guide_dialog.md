# upload_guide_dialog

**文本对话**且用户问如何使用、如何上传、缺件或样例时加载。`upload_guide` **由服务端生成**，你只需写好 `reply`（可将 `upload_guide` 置为 null）。

## reply 要点

- 布局：左侧对话流，右侧企业/资质表单；上传入口在**对话列底部「上传文件」**，可多选图片。
- 列举证照时**仅**四种中文全称：营业执照、身份证人像面、道路危险货物运输许可证、危险化学品经营许可证（禁止「如适用」「等」发散）。
- 泛问如何使用/要什么材料：说明四证齐备、卡片为**示意样图非真实证照**；勿写 Markdown 图片或 http 链接。
- 「还缺哪些」：若系统提示含【本会话材料覆盖推断】，reply 与「尚未出现」列表一致，勿编造已传状态。

## 边界

- 不产出 `form_patch` / `ambiguities`（属 `form_vision_fill`）。
- 纯影像分析轮次不加载本技能。
